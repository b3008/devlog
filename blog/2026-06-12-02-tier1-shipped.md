2. **Exit-0 Stop hook** (`stop.py` both copies): `sys.exit(2)` → `sys.exit(0)` with the same block JSON. The red "Stop hook error" box is gone for good, not worked around.
3. **Global path fix**: `_install_global` now writes `~/.claude/CLAUDE.md` as the README always claimed, and migrates: a legacy `~/CLAUDE.md` gets its sentinel stripped (deleted if devlog-only). 4. **Thin local block**: local installs that detect an active global install inject a ~90-word pointer block instead of the full ~1.5k-token convention. `--full` overrides for repos with collaborators who lack the global install. This repo's own CLAUDE.md is now the thin block.
5. **Hook manifest hygiene**: `manifest.hooks` records carry `sha256`; reinstalls carry hooks forward instead of orphaning them (which also resyncs stale scripts — the untrimmed global reminder fixed itself this way); customized scripts are detected and preserved on both reinstall and uninstall.
6. **Runtime hook dedup**: the global hook instance reads the project's `.claude/settings.json` and defers when a project-local devlog hook is registered. One reminder per stop, no uninstalls required.
7. **`devlog index`**: `_index.md` is now generated from entry frontmatter (newest first, heading preserved). Convention step 6 and `/devlog-write` say "run `devlog index` if available" with hand-editing as fallback.
8. **SessionEnd coverage**: `--with-hook` now installs a bundle — the Stop reminder plus a `session_end.py` recorder appending one JSON line per session to `.devlog/sessions.jsonl`. `devlog status` reports `N recorded, last DATE — K since the last entry`.

Plus the convention text changes from the assessment: step 7 (annotate superseded entries in the same turn), step 8 (commit the entry with the work), and media defaults that ask for artifacts terminal agents can actually produce. Tests: 104 → 140, all green.

## Why it matters

This closes every "capture is polite, consumption doesn't scale, concurrency breaks things" gap the assessment rated as small-diff fixable. The standing per-session tax on this machine drops from two full convention copies plus two competing reminders to one full copy, one thin pointer, one reminder, and a coverage line in a log. And devlog finally measures itself: `sessions.jsonl` turns "does the convention fire?" from a feeling into a number, which is the explicit gate on building the smarter mutation-aware hook.

The deeper pattern: most of these bugs were *scope-interaction* bugs — global and local installs were each fine alone and broken together. The fixes deliberately pair a static dedup (thin block, one convention) with a dynamic one (runtime defer, one reminder), because installs can happen in either order and neither side can assume the other's state at install time.

## How it works

Two designs worth recording:

- **Global-defers-to-local, not the reverse.** The convention dedup puts the full text globally and a pointer locally; the hook dedup runs the other way — the global script checks `<cwd>/.claude/settings.json` for a project-local devlog hook and exits silently if one exists. Asymmetric on purpose: context should be injected once from the broadest scope, but runtime behavior should be owned by the most specific scope. The check is best-effort with every failure path falling through to "fire" — a broken settings file must never silence the reminder.
- **Carry-forward as drift repair.** Reinstalls now treat hooks like commands: hash on record, preserve if the user diverged, otherwise resync from the template. The carry-forward rule (a reinstall without `--with-hook` refreshes existing hooks rather than forgetting them) is what retroactively fixed this machine's stale global script — the drift the 06-11 assessment observed was repaired by the mechanism built in response to it.

## Surprises

- The "six weeks of unshipped work" was half-illusion: PR #11 had been squash-merged remotely and only the local `main` was stale. The genuinely unshipped part was the working tree — which still proves the finding (nothing in the convention said "commit the entry"), but the diagnosis sharpened: the gap is between working tree and *any* branch, not between branch and main.
- The test suite turned out to be non-hermetic in a way nobody had hit: thin-block detection reads `Path.home()`, so the suite would have passed differently on machines with a real global devlog install. An autouse `isolated_home` fixture fixed it — but it's another instance of the project's recurring theme that every new capability quietly adds a new failure surface.
- Same theme, sharper edge: the SessionEnd subprocess tests initially inherited the test runner's own environment — including the `CLAUDE_PROJECT_DIR` this very session runs under. Without stripping it, the tests would have appended fake session records to *this repo's* real `sessions.jsonl`. The tool's tests nearly polluted the tool's own dogfood data.
